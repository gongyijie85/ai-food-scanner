# 领域上下文（Domain Context）

## 项目概述

**拍了就懂** 是一个面向老年人的中国预包装食品/保健食品标签识别工具。
用户上传配料表照片，AI 识别配料与添加剂，系统根据 GB 2760 标准和个人健康档案给出风险提示与语音播报。

## 核心领域术语

| 术语 | 英文 | 含义 |
|------|------|------|
| 配料表 | Ingredient List | 预包装食品包装上按添加量降序排列的全部原料 |
| 食品添加剂 | Food Additive | GB 2760 标准中允许使用的添加剂（如碳酸氢钠、大豆磷脂） |
| 保健食品 | Health Supplement | 带「蓝帽子」标志、批准文号/备案号的特殊食品 |
| 风险评分 | Risk Score | 基于添加剂风险等级与健康档案计算出的 0-100 分数 |
| 健康档案 | Health Profile | 用户的疾病、过敏原、用药信息 |
| 人群标签 | Health Group | 从健康档案提取出的关注人群（糖尿病、高血压、脑梗/心血管、减脂、过敏、儿童、孕妇） |
| 药物-食物冲突 | Drug-Food Conflict | 用户用药与食品成分之间的潜在相互作用 |
| 过敏原 | Allergen | 可能引发过敏反应的配料 |
| 识别结果 | Scan Result | AI 返回并经过本地归一化后的食品/保健品解析结果 |
| 配料表区域 | Ingredient List Region | 包装上"配料表"字样及其后续文字所在的局部区域，识别时应优先聚焦于此 |
| AI 推断 | AI Inference | 模型根据常识、图片背景或常见配方补全，但未在包装配料表原文中出现的配料 |
| 历史记录 | History | 本地保存的过往识别结果 |

## 架构定位

- `utils/api.py`：外部 AI 模型交互与模型输出归一化
- `utils/score.py`：添加剂风险判定、评分计算、健康建议生成
- `utils/data.py`：本地静态数据加载（GB 2760、过敏原、疾病、药物冲突）
- `utils/history.py`：本地历史记录读写
- `utils/helpers.py`：页面路由、设备检测等通用工具
- `utils/security.py`：安全转义
- `pages/`：Streamlit 页面渲染
- `components/`：可复用 UI 组件

## 关键约束

- API 端点使用 MiMo 新加坡集群
- 图片压缩后 base64 不超过 2MB
- 结果中 `product_name` 必须为中文
- 添加剂判定以 GB 2760 为准
- 所有 UI 动态文本必须经过 `_safe()` 转义
