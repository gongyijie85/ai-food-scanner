# Domain docs

## Layout

This repo uses a **single-context** layout:

- Domain language and project overview: `CONTEXT.md` at the repo root.
- Architectural Decision Records (ADRs): `docs/adr/<slug>.md`.

## Reading rules

- Before proposing refactors or diagnosing bugs, read `CONTEXT.md` first.
- Before making architectural decisions, check existing ADRs in `docs/adr/`.
- If a decision changes or contradicts an existing ADR, create a new ADR and reference the old one.
