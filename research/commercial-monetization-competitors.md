## 1. 研究问题与范围

**问题**：国内「食品配料/添加剂查询、拍照识配料、健康扫码」类产品如何变现、价格带大概在哪、主要卖给谁？

**范围（优先公开可查）**：

- App Store / 应用宝等头部相关 App（配料查查、知了、成分查查、成分喵、有食问问、料拍拍等）
- 大平台附带能力（薄荷健康「看懂配料表」等）
- 国际对标（Yuka）与零售侧「配料干净」趋势（说明 B 端需求，不等于扫码 App 的 B 端收入）
- 会员价、广告、电商导流、是否存在明确 B 端报价

**不在范围**：未实现的产品功能设计；未经 App 内实测的转化率；非公开的 GM 拆账。

## 2. 变现模式全景

赛道内可归纳为 **五层变现**（由轻到重）：

| 层级 | 模式 | 典型做法 | 在本赛道出现情况 |
|------|------|----------|------------------|
| L1 | 免费获客 + 次数/功能墙 | 首 1–几次扫码免费，深度解读/无限次锁定 | 极常见；用户差评也集中在「查一次就要会员」 |
| L2 | 订阅会员（月/季/年/永久） | IAP 连续包月/年；部分有日卡、买一年送一年 | **纯配料工具的主路径** |
| L3 | 信任叙事（少广告/不接广告） | 文案强调「不接品牌广告」；用会员补贴成本 | 知了、Yuka、成分喵等强调；与付费并存 |
| L4 | 电商/商城/导流 | App 内健康零食商城、「去购买」跳转旗舰店 | 知了上线商城；成分喵有购买入口；薄荷自有食品 |
| L5 | 平台化闭环 | 会员 + 自有食品 + 互联网医疗 + B 端 SaaS | 仅薄荷健康等大体量玩家；**不是小工具起步形态** |

**观察**：

1. **纯「扫码/拍照查配料」工具几乎全是 C 端 freemium 订阅**，很少看到公开的企业套餐价或 API 价目表。
2. **广告不是主叙事**：多款产品用「无广告/客观中立」做信任点；真正的收入在订阅与电商。
3. **大平台（薄荷）把配料解读当增值模块**，钱主要来自体重管理会员、食品商城、问诊与 B 端减重 SaaS，而非单独卖「查添加剂」。
4. **零售侧「配料干净」是另一条价值链**：商超/会员店用清洁标签提升选品与会员粘性；这验证了市场需求，但付费方是零售与品牌，不是扫码 App 用户。

| 产品 | 价格（公开） | 变现结构 | 客群 |
|------|--------------|----------|------|

| **Yuka**（国际） | **€15 / 年**（约 ¥110–¥120） | 免费扫码 + 低价年费解锁离线/搜索/偏好/无限历史；强调独立、无广告扭曲评分 | 健康意识消费者（欧美为主） |

### 3.3 价格带小结（给决策用）

```
纯配料工具年费锚点：  ¥68 ──────── ¥99
                        配料查查      成分查查
                        知了~¥78

体重管理平台年费：     ~¥128（薄荷）
国际信任型扫码：       ~€15/年（Yuka）
「全家/慢病」溢价试探： 目前公开 SKU 很少单独卖「家庭套餐」；
                      料拍拍 ¥198 年费偏孤立，不宜当主流锚点
```

**可支付意愿粗判（非严谨调研）**：成分党对 **¥70–¥100/年** 的工具订阅已形成心智；更高价需要可感知的「慢病/家庭/专业」增量，否则易被卸载。

## 4. 主要卖给谁

### 4.1 画像（从商店描述、评论与行业报道归纳）

| 客群 | 动机 | 谁在服务 | 付费意愿线索 |
|------|------|----------|--------------|
| **成分党 / 年轻消费决策者** | 「科技与狠活」、0 添加、看懂添加剂 | 几乎所有配料 App | 订阅主力；对广告与过早付费墙敏感 |
| **宝爸宝妈 / 儿童零食选购** | 儿童是否适合、糖盐添加剂 | 知了、有食问问、成分查查等写进卖点 | 中高；家庭决策场景 |
| **减脂/控糖/健身** | 热量 + 配料干净 | 薄荷为主；配料 App 做标签 | 薄荷闭环更强 |
| **过敏 / 孕期 / 部分慢病（痛风、三高、糖尿病）** | 个性化风险提示 | 成分查查、有食问问、料拍拍有标签 | 有「试用后月订」的真实评论；**尚未形成「用药冲突」专业品类** |
| **化妆品成分党** | 一站查食品+美妆 | 成分喵、美丽修行等 | 拉高 DAU 与电商导流 |
| **银发本人 / 子女代际关怀** | 看不清、看不懂、药食是否冲突 | **公开头部产品几乎未主打适老化、语音播报、家属共享** | **空白；本项目假设待验证** |

### 4.2 谁付钱（Payer）vs 谁使用（User）

| 模式 | 谁用 | 谁付 | 赛道现状 |
|------|------|------|----------|
| 自用自付 | 年轻人/家长 | 本人 | **默认模式** |
| 子女付 / 家庭账号 | 老人用 | 子女付 | 公开 SKU 罕见；适合「拍了就懂」验证 |
| B 端采购 | 机构用户 | 医院/企业/品牌 | 薄荷有减重 SaaS；纯配料工具未见成熟 B 报价 |
| 零售选品赋能 | 导购/会员 | 零售商 | 「配料干净」在山姆、叮咚、1 号会员店等，**不是扫码 App 收入** |

### 4.3 适老化 / 慢病 / 用药切口占位判断

| 能力 | 头部配料 App | 备注 |
|------|--------------|------|
| 大字/语音/一步操作 | 基本未见主打 | 适老化是差异化窗口 |
| 慢病标签（痛风/糖/三高） | 有文案与部分规则 | 多为标签提示，非档案深度 |
| **药物–食物冲突** | **几乎未见产品级占位** | 与服药助手类 App 割裂 |
| 家属共享 / 远程关怀 | 罕见 | 知了等有报告向，但非「子女–老人」闭环 |
| 保健品辅料解读 | 弱 | 老人场景潜在需求 |

**结论**：慢病「标签化」有人做；**「适老化体验 + 用药冲突 + 家属付费」组合在公开市场上仍基本空白**——这是机会，也意味着 **没有现成定价可抄，必须用真实用户验证**。

## 6. 对「拍了就懂」的 3 条启示

### 启示 1：定价先对齐「工具年费锚点」，不要对标薄荷平台价

- 市场已教育：**纯解读工具年费约 ¥68–¥99**；月费 ¥10–¥28。
- 验证期不必上完整支付系统，但访谈/试用应挂上 **可感知的价格锚**（建议假设：**个人年卡 ¥79 或 家庭年卡 ¥99–¥129**），专门测「谁付、付多少、为什么」。
- 避免：首扫即强付费墙（竞品差评重灾区）；也避免无价格谈「愿不愿意付」（答案会虚高或虚低）。

### 启示 2：默认付费方更可能是「子女/家属」，而非与成分党抢同一 SKU

- 现有产品把付费方默认成 **会自己下载、会看订阅页的年轻人**。
- 「拍了就懂」的差异化（适老化、语音、健康档案、药物冲突）天然偏向 **User=老人/慢病者，Payer=子女**。
- 验证应拆开问：
  - 老人是否愿意反复用？
  - 子女是否愿意为父母付 ¥80–¥130/年？
  - 家庭套餐是否比个人订阅更易成交？
- **在未验证前，不要把 B2B 或老人自付当作事实**（与 #26 一致）。

### 启示 3：空白在「药食冲突 + 适老体验」，不在「再做一个订阅墙」

- 订阅、扫码、添加剂百科、儿童/控糖标签 **已经红海**；再做一个通用配料 App 只能陷入 ¥9.9 月费价格战。
- 公开市场 **几乎无人**把「用药冲突 + 大字语音 + 家属共享」做成产品级卖点——这是差异化，也是合规敏感区（表述须像健康信息工具而非医嘱，交叉 #32）。
- 变现路径建议优先级（验证期）：
  1. **C 端家庭订阅假设**（子女付）  
  2. 中长期才考虑健康零食导流 / 机构合作（对标知了商城、薄荷 SaaS）  
  3. 不要为变现提前建支付后台与完整商城（#26 Out of scope）

## 7. 给 #26 地图的决策输入（非最终决策）

| 议题 | 研究给出的输入 | 仍须验证 |
|------|----------------|----------|

| 价格带 | 工具年费 ¥68–¥99；月 ¥10–¥28 | 适老/家庭溢价能否站上 ¥99–¥129 |

| B 端 | 仅大体量玩家有 SaaS；零售「配料干净」是趋势非 App 收入 | V1 不依赖 B 端 |
| 功能冻结 | 验证期勿堆商城/广告 SDK | 与 #29 冻结清单对齐 |
