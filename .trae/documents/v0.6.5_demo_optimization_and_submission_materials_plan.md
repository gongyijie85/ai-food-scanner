# v0.6.5 参赛 Demo 体验优化与资料补全计划

## Summary

为让评委在初赛中 3 秒内进入核心功能，计划在 `app.py` 增加 `?demo=1` 评委快速模式：自动完成法律同意、跳过 4 步引导、预填默认健康档案，并在该模式下隐藏侧边栏模型切换与扫描页输入方式单选。同时同步更新 `README.md`、本地 `初赛Demo帖` 草稿的版本号、公开链接、父亲病史、Session ID 与使用步骤，并补充 v0.6.5 变更日志。

## Current State Analysis

- **入口流程过重**：[app.py](file:///d:/GBT/ai-food-scanner/app.py#L158-L169) 首次访问强制先走法律同意页，再走 4 步引导页，评委打开链接后不能立即看到首页。

- **扫描页多一步选择**：[pages/scan.py](file:///d:/GBT/ai-food-scanner/pages/scan.py#L127-L134) 每次都要在“拍照 / 从相册选择”间手动切换。

- **Demo 帖草稿滞后**：[d:\GBT\初赛Demo帖_AI食品配料表识别工具.md](file:///d:/GBT/初赛Demo帖_AI食品配料表识别工具.md) 中版本为 v0.5.7、父亲病史写“脑梗 8 年”、使用步骤与实际流程不符、最新更新缺少 v0.6.x、Session ID 全是 6 月。

## Proposed Changes

### 1. `app.py`：增加 `?demo=1` 评委快速模式

**文件**：[app.py](file:///d:/GBT/ai-food-scanner/app.py)

**What/Why**：评委打开 `.../?demo=1` 后直接进入首页，避免法律同意和引导页打断演示；普通用户访问仍走完整流程。

**How**：
- 新增辅助函数 `_apply_demo_mode()`，在 `main()` 入口处检测 `st.query_params.get("demo") == "1"`。
- 若命中：
  - `st.session_state["legal_agreed"] = True`
  - `st.session_state["onboarded"] = True`
  - `st.session_state["demo_mode"] = True`
  - 预填 `health_profile = {"diseases": ["脑梗/心血管", "高血压"], "age": 60, ...}`
- 保持法律页和引导页代码不变，仅通过 session state 跳过。

### 2. `components/navigation.py`：评委模式下隐藏模型切换

**文件**：[components/navigation.py](file:///d:/GBT/ai-food-scanner/components/navigation.py)

**What/Why**：MiMo 是主模型，Agnes 仅降级备用，评委不需要看到切换入口。

**How**：
- 在 `render_desktop_sidebar()` 中，将“高级设置” expander 用 `if not st.session_state.get("demo_mode"):` 包裹。
- 法律声明 expander 保留，供评委随时查看合规说明。
- 移动端底部导航无模型切换，无需改动。

### 3. `pages/scan.py`：评委模式下自动选择输入方式

**文件**：[pages/scan.py](file:///d:/GBT/ai-food-scanner/pages/scan.py)

**What/Why**：减少评委在扫描页的一次无意义点击。

**How**：
- 检测 `st.session_state.get("demo_mode")`。
- 若为 demo 模式：
  - 手机端默认 `input_method = "拍照"`
  - 桌面端默认 `input_method = "从相册选择"`
  - 不渲染 `st.radio` 单选组件。
- 非 demo 模式保持现有交互不变。
