# 首页、引导页、扫描页重新设计计划

## 摘要

根据用户反馈和确认，对 AI 食品配料表识别工具进行三项核心界面改造：

1. **引导页标签统一**：将引导页"您属于哪类人群"的选项与 health_profile 疾病列表对齐，解决标签不对/重复问题。
2. **首页重新设计**：从"单一大扫描按钮"改为"历史记录列表 + 底部双按钮（拍照 / 健康档案）"，信息密度更高，操作更符合中老年用户习惯。
3. **扫描页交互重构**：改为"相机优先"模式，进入页面直接展示取景框并提示"对准商品自动识别"，下方提供相册和最近识别入口，简化当前混乱的多入口问题。

所有改动先通过 HTML 预览稿确认，再同步到 Streamlit 代码。

---

## 当前状态分析

### 1. 引导页标签问题

当前 `utils/constants.py` 中 `CONDITION_ITEMS` 定义了 7 个选项：

```python
CONDITION_ITEMS = [
    ("diabetes", "糖尿病", "🩺"),
    ("hypertension", "高血压", "🫀"),
    ("stroke", "脑梗/心血管", "🧠"),
    ("fat-loss", "减脂", "🥗"),
    ("allergy", "过敏", "🤧"),
    ("children", "儿童", "🧒"),
    ("pregnant", "孕妇", "🤰"),
]
```

而 `pages/onboarding.py` 和 `pages/profile.py` 实际展示疾病时使用的是 `DISEASE_ICONS` 映射：

```python
DISEASE_ICONS = {
    "脑梗/心血管": "❤️",
    "糖尿病": "🩸",
    "高血压": "🫀",
    "痛风": "🦴",
    "乳糖不耐": "🍼",
    "肾病": "🌾",
}
```

导致引导页出现：
- 糖尿病/高血压/脑梗/心血管 → 能匹配到对应图标
- 减脂/过敏/儿童/孕妇 → 匹配不到，显示默认 `🏷️`
- 用户感觉"标签不对"、"有重复"

### 2. 首页问题

当前 `pages/home.py` 采用：
- 顶部标题 + 副标题 + 右侧心形健康档案入口
- 健康标签横向滚动
- 中央巨大圆形"扫描配料表"按钮
- "点击大按钮开始"提示气泡与按钮重叠（用户已截图反馈）
- 底部免责声明

用户希望：
- 直接展示历史记录，方便快速回看
- 底部固定两个按钮：拍照、健康档案
- 健康标签不再首页展示，放进档案页

### 3. 扫描页问题

当前 `pages/scan.py` 同时出现：
- 取景框展示区（纯视觉）
- `st.file_uploader` 文件上传器
- `st.camera_input` 相机输入
- "拍照"按钮（只能 toast 提示用户点上面的相机）
- "从相册选择"按钮（只能 toast 提示用户点上面的上传器）

交互逻辑混乱，用户需要明确：
- 进入扫描页 = 直接拍照
- 取景框内提示"对准商品自动识别"
- 下方有相册/最近识别入口

## 改动方案

### Phase 1：HTML 预览确认

先创建两个独立 HTML 预览文件，放到 `design/` 目录：

| 文件 | 用途 |
|------|------|
| `design/home_v2_preview.html` | 新首页：标题 + 最近 3 条历史 + 底部拍照/档案双按钮 |
| `design/scan_v2_preview.html` | 新扫描页：取景框 + "对准商品自动识别" + 相册/最近识别入口 |

预览文件使用现有设计系统（绿色主色、圆角、卡片阴影），便于用户直观确认。

### Phase 2：代码改造

#### 2.1 统一引导页标签

**涉及文件**：
- `utils/constants.py`
- `pages/onboarding.py`
- `pages/profile.py`

**具体改动**：

1. 修改 `utils/constants.py` 中的 `CONDITION_ITEMS`，使其与档案页疾病列表一致：

```python
CONDITION_ITEMS = [
    ("stroke", "脑梗/心血管", "❤️"),
    ("diabetes", "糖尿病", "🩸"),
    ("hypertension", "高血压", "🫀"),
    ("gout", "痛风", "🦴"),
    ("lactose", "乳糖不耐", "🍼"),
    ("kidney", "肾病", "🌾"),
]
```

2. 删除 `pages/onboarding.py` 中独立的 `DISEASE_ICONS`，改为直接使用 `CONDITION_ITEMS` 第三个字段的 emoji。

3. 默认预选疾病保持为 `["脑梗/心血管", "高血压"]`，与档案页默认一致。

4. `pages/profile.py` 的疾病网格也统一使用 `CONDITION_ITEMS`，删除重复的 `DISEASE_ICONS` 映射（如已在多处定义，则集中到一个文件）。

#### 2.2 首页重新设计

**涉及文件**：
- `pages/home.py`
- `components/top_nav.py`
- `.streamlit/style.css`

**新布局**：

```
┌─────────────────────────┐
│ 食品配料表识别          │  ← 标题
│ 拍照即懂，吃得更安心    │  ← 副标题
├─────────────────────────┤
│ 最近识别                │  ← 小标题
│ ┌─────┐                 │
│ │ 85  │ 鲜牛奶        │  ← 历史卡片 1
│ └─────┘ 1种添加剂      │
├─────────────────────────┤
│ ┌─────┐                 │
│ │ 72  │ 某品牌饼干    │  ← 历史卡片 2
│ └─────┘ 3种添加剂      │
├─────────────────────────┤
│ ┌─────┐                 │
│ │ 45  │ 某品牌辣条    │  ← 历史卡片 3
│ └─────┘ 7种添加剂      │
├─────────────────────────┤
│ [     查看全部历史    ] │  ← 文字按钮/链接
├─────────────────────────┤
│ [  📷 拍照识别  ]       │  ← 底部大按钮 1
│ [  ❤️ 健康档案  ]       │  ← 底部大按钮 2
├─────────────────────────┤
│ AI识别仅供参考...       │  ← 免责声明
└─────────────────────────┘
```

**具体改动**：

1. `pages/home.py`：
   - 保留 `render_top_nav(..., show_back=False)`，但移除 `right_action="profile"`（健康档案入口改到底部按钮）。
   - 删除健康标签区域。
   - 删除中央大圆形扫描按钮和提示气泡。
   - 新增"最近识别"区域，调用 `utils.history.load_history()` 读取最近 3 条记录。
   - 每条历史记录显示：左侧圆形评分徽章（颜色对应安全/注意/高风险）、产品名、添加剂数 + 日期。
   - 点击历史卡片跳转到详情页。
   - 底部两个大按钮：
     - "📷 拍照识别" → `switch_page("scan")`
     - "❤️ 健康档案" → `switch_page("profile")`

2. `components/top_nav.py`：
   - 保持现有能力，但首页调用时不再传 `right_action`。

3. `.streamlit/style.css`：
   - 新增/调整首页历史卡片样式、底部双按钮样式。
   - 移除或调整与大圆形扫描按钮相关的 CSS。

#### 2.3 扫描页重构

**涉及文件**：
- `pages/scan.py`
- `.streamlit/style.css`

**新布局**：

```
┌─────────────────────────┐
│ ←  扫描识别             │  ← 顶部导航
├─────────────────────────┤
│ 对准商品自动识别        │  ← 提示文字
├─────────────────────────┤
│ ┌─────────────────────┐ │
│ │  [    取景框      ] │ │  ← 相机输入区
│ │  [    扫描线      ] │ │
│ └─────────────────────┘ │
├─────────────────────────┤
│ [  🖼️ 从相册选择  ]     │  ← 相册入口
│ [  📜 最近识别    ]     │  ← 最近识别入口
├─────────────────────────┤
│ 提示：请正对配料表...   │
└─────────────────────────┘
```

**具体改动**：

1. `pages/scan.py`：
   - 顶部保留 `render_top_nav("扫描识别", back_target="home")`。
   - 提示文字从"对准包装上的配料表"改为"对准商品自动识别"。
   - 取景框区域直接放置 `st.camera_input`，而不是单独放取景框 SVG + 下面再放相机组件。
   - 保留取景框角标和扫描线动画作为视觉叠加（通过 HTML + CSS 覆盖在相机输入区上方）。
   - 用户拍照后（`camera_input` 返回值非空），自动进入识别流程，无需再点"使用照片"。
   - 下方提供两个次要入口：
     - "从相册选择"：使用 `st.file_uploader`（不再隐藏，直接展示为一个按钮样式或简化上传器）。
     - "最近识别"：展示最近 2-3 条历史记录缩略图，点击可跳转详情/重新识别。
   - 删除当前混乱的"拍照"按钮和"从相册选择"按钮（因为相机输入本身就能拍照）。

2. `.streamlit/style.css`：
   - 调整扫描页取景框覆盖在 `st.camera_input` 上的样式。
   - 调整相册/最近识别入口样式。

**技术注意点**：
- Streamlit 的 `st.camera_input` 无法通过程序自动打开相机快门，必须由用户点击。因此"进入页面直接唤起相机"在 Streamlit 中实际表现为：页面一加载就显示相机组件，用户看到取景画面，点击快门即可拍照。
- 拍照后组件返回图片文件，代码检测返回值并自动调用 `_scan_validate_and_recognize`。

#### 2.4 设计稿同步策略

用户选择"只同步核心视觉"，因此：
- 不强制复刻 HTML 预览稿中的每一个动画细节。
- 重点同步：颜色系统、圆角、卡片阴影、按钮形状、图标风格、间距比例。
- 保持现有 Streamlit 组件结构，通过 CSS 覆盖实现视觉统一。

## 假设与决策

| 决策 | 理由 |
|------|------|
| 引导页只保留疾病类标签 | 用户选择与档案页统一，避免减脂/过敏/儿童/孕妇等非疾病标签混淆 |
| 首页不显示健康标签 | 用户明确选择"放进档案入口"，首页保持清爽 |
| 历史记录显示最近 3 条 | 用户确认，兼顾信息密度和一屏可见性 |
| 底部双按钮固定 | 符合中老年用户操作习惯，拍照和档案两大核心功能随时可点 |
| 扫描页相机优先 | 用户确认，进入页面即可看到相机画面，减少一步点击 |
| 先出 HTML 预览再改代码 | 用户明确要求，降低返工风险 |
